1 · Introducción y arquitectura

YPVA Admin es un panel de gestión profesional para fotógrafos, construido específicamente para yezraelperez.es. Reemplaza la librería de medios estándar de WordPress con una experiencia móvil-first que añade carpetas, IA, galerías privadas para clientes, mapa de ubicaciones, slideshow profesional y mucho más.

Tres piezas acopladas

  • Plugin yz-media-folders en el WordPress de yezraelperez.es. Aporta carpetas, REST API, IA, mapa, geolocalización, bridge con portfolios del tema kotlis, y los servicios de marca/seguridad.
  • PWA en app.yezraelperez.es: cliente Vue 3 que consume la REST API. Instalable como app en móvil/desktop. Funciona offline parcialmente.
  • Plugins satélite: yzmf-client-portal (galerías privadas con token), yzmf-hotlink-watermark (marca de agua en hotlinking).

Modos de uso

La PWA está pensada para usar a diario desde el móvil, pero todas sus pantallas se adaptan a tablet y escritorio (con sidebar lateral en pantallas anchas). El plugin sigue accesible desde wp-admin para tareas que requieran herramientas avanzadas de WordPress (editar HTML crudo, gestionar plugins, usuarios complejos, etc.).

Filosofía El 95% del trabajo del día a día se hace en la PWA. El wp-admin queda para cosas que pasan rara vez: instalar/actualizar plugins, ajustes globales del tema, etc.

2 · Inicio de sesión

La pantalla de login ofrece dos modos que se eligen con tabs en la cabecera:

🔑 Usuario + contraseña (modo simple, por defecto)

La contraseña habitual de tu usuario WordPress. Es la opción más rápida para empezar. Funciona gracias al handler de Basic Auth integrado en el plugin (class-basic-auth.php) que permite usar contraseñas regulares en la API REST.

🔐 Application Password (modo avanzado, recomendado para uso continuo)

Token de 24 caracteres generado en tu perfil de WordPress. Aporta:

  • Si cambias tu contraseña principal, la sesión sigue válida.
  • Puedes revocar cada token individualmente sin afectar otros dispositivos.
  • Cada dispositivo o app puede tener el suyo con nombre identificativo.
  • No funciona para iniciar sesión en wp-admin; sólo para la API. Si te lo roban, tu cuenta principal sigue protegida.

Crear una Application Password

  1. Login en la PWA con tu contraseña normal.
  2. Ajustes → Usuarios → tu usuario → sección "Application Passwords".
  3. Botón "+ Nueva", ponle un nombre (ej. "iPhone PWA", "Lightroom").
  4. Copia la contraseña generada — solo se muestra una vez.
  5. Cierra sesión y vuelve a entrar con el modo "🔐 Application Password".

Otros campos del login

  • URL del sitio: por defecto https://yezraelperez.es. Solo cámbialo si conectas a otro WordPress.
  • 👁 mostrar/ocultar contraseña: para verificar lo que tecleas, especialmente con tokens largos.
Bloqueo por intentos fallidos Tras 5 intentos fallidos en 15 minutos, tu IP se bloquea durante 30 minutos. Configurable en Ajustes → Política de bloqueo. Si te bloqueas a ti mismo, puedes liberar IPs desde otro dispositivo o desde wp-admin.

3 · Pantalla principal: Dashboard

La pantalla de inicio es un panorama visual de tu catálogo. Pull-to-refresh (deslizar hacia abajo) refresca todos los widgets a la vez.

Widgets de un vistazo

  • Saludo + KPIs: Medios · Portfolios · Ubicaciones · Carpetas. Click en cada uno te lleva a su pantalla.
  • Últimas subidas: tira con las 12 fotos más recientes. Click → MediaDetail.
  • Heatmap calendario: tipo GitHub. Cada celda es un día del último año, intensidad según subidas. Etiquetas de mes y total + racha actual de días consecutivos.
  • Salud del catálogo: anillo SVG con porcentaje 0-100% (verde/azul/ámbar/rojo) basado en alt + geo + tags + carpeta. Lista de tareas pendientes con click directo a la lista filtrada.
  • Mini-mapa: preview Leaflet con últimas 120 fotos georreferenciadas. Click abre el Mapa completo.
  • Almacenamiento: GB usados + total imágenes + subidas en 30 días.
  • Top carpetas: las 5 con más imágenes.
  • Top tags IA: chips con las 12 etiquetas más comunes; click filtra Media.
  • Acciones rápidas: subir, gestionar carpetas, nuevo portfolio, galerías cliente, etc.
  • Paleta global: 24 colores dominantes del catálogo entero. Click filtra Media por ese color.
Caché del Dashboard Las estadísticas se cachean 5 minutos en el servidor para no machacar la BD. Si acabas de subir fotos y los números no cambian, espera o haz pull-to-refresh en bucle (cada refresh fuerza la regeneración después del cache).

La pantalla central de la app. Aquí está todo tu catálogo con filtros, búsqueda, selección múltiple y acciones masivas.

Toolbar superior

  • Búsqueda: por título, alt, caption. Búsqueda especial: __NO_ALT__ filtra imágenes sin alt text.
  • Filtros rápidos: 🆕 Hoy · ⚠ Sin alt · 📭 Sin carpeta · 🖼 Solo fotos · 🎬 Solo vídeos · 📦 Más pesadas.
  • 🏷 Tag y 🎨 Color: filtros por etiqueta IA o color dominante.
  • Orden: fecha / nombre / tamaño + dirección ascendente/descendente.
  • Vista: ⊞ rejilla o ≡ lista (con info extendida).
  • ▶ Slideshow: lanza el modo presentación con la selección actual.

Modo selección y acciones masivas

Editor de títulos en lote ✏

El bottomsheet ofrece 8 operaciones distintas con previsualización en tiempo real:

  • 🔁 Buscar y reemplazar: literal o regex, con opción case-sensitive.
  • ◀ Añadir prefijo: añade texto al inicio (ej. "Boda María — ").
  • ▶ Añadir sufijo: añade texto al final (ej. " (© Yezrael)").
  • # Numerar: patrón con {n}, padding con ceros, orden por fecha o título. Ej. Boda María #{n} con padding 3 → "Boda María #001".
  • 📄 Desde nombre: pone como título el nombre del archivo en disco. Opciones para quitar extensión y convertir _/- en espacios.
  • 🤖 Desde alt: usa el alt text como título. Útil después de generar metadatos con IA.
  • Aa Mayúsculas: Title Case · "Solo la primera" · todo minúsculas · TODO MAYÚSCULAS.
  • ⌫ Limpiar espacios: colapsa espacios duplicados y los del principio/final.
Workflow recomendado tras una sesión 1. Sube las fotos → 2. Selecciona todas las de la sesión → 3. Acción ✨ para generar IA → 4. Acción ✏ con "🤖 Desde alt" para que los títulos sean descriptivos → 5. Otra acción ✏ con "Numerar" si quieres un código de sesión consistente.

5 · Edición individual de imágenes

Click en cualquier foto abre MediaDetail. Layout adaptable: móvil apilado, desktop a 2 columnas (foto a la izquierda sticky, panel a la derecha).

Cabecera (zona de la foto)

  • ‹ ›: navegar a la anterior/siguiente foto (también con flechas del teclado).
  • Contador: posición / total dentro de la lista actual.
  • Swipe: en móvil, deslizar el dedo izq/der cambia de foto.
  • Atajos: ⬇ Descargar (fetch + blob, fuerza descarga real con nombre original) · 📤 Compartir (Web Share API o copy URL fallback) · 📋 Copiar URL · ↗ Abrir original.

Tab "Datos"

  • Título, alt, SEO title, caption, descripción.
  • Carpeta: bottomsheet para cambiarla.
  • Etiquetas IA: chips clicables que filtran Media por ese tag.
  • ✨ Generar alt+caption: llama Claude con la imagen y rellena los campos automáticamente.
  • Guardar: persiste todos los cambios.

Tab "📍 Ubicación"

Si la imagen tiene GPS (extraído del EXIF o asignado a mano), mini-mapa con marker. Botón "📍 Cambiar" abre un GeoTagger con búsqueda Nominatim.

Tab "EXIF"

Tarjetas con todos los metadatos extraídos: cámara, focal, apertura, ISO, velocidad, fecha, dimensiones, peso, tipo MIME, archivo original.

Tab "⚒ Herramientas"

  • 📋 Copiar URL pública.
  • ⚙️ Abrir en el editor de WordPress (link directo a wp-admin).
  • 🗑 Eliminar la imagen (con confirmación).

6 · Subir fotos

Elegir origen

  • 📷 Hacer foto: en móvil abre la cámara directamente (capture="environment").
  • 🖼 Galería: explorador de archivos del sistema. Permite multi-selección.
  • Carpeta destino: dropdown con árbol — selecciona dónde van las fotos. "— Sin carpeta —" para subir sueltas.

Cola de subida

Cada archivo aparece como tarjeta con thumbnail (si es imagen), nombre, tamaño y estado: ⏳ pendiente · ⚪ subiendo · ✓ ok · ✕ error.

  • Botón "↑ Subir N archivos" procesa la cola en serie.
  • Tras subir cada imagen, la PWA extrae la paleta dominante en cliente (Canvas + cuantización a 5 bits/canal) y la guarda en el servidor.
  • Si tiene GPS en EXIF, se geolocaliza automáticamente.
  • "Limpiar OK" elimina las completadas, "Vaciar" descarta todas.
Subir en otro idioma La cola es secuencial (no paralela) para no saturar el servidor compartido. Para sesiones grandes, usa el watcher de Lightroom (ver sección 16).

8 · Portfolios

Los portfolios son entradas del CPT portfolio del tema kotlis. La PWA los edita íntegros (galería, layout, categorías, configuración avanzada del tema) sin necesidad de tocar wp-admin.

Listado

Búsqueda + filtro por categoría + paginación. Cada tarjeta muestra título, layout, estado, miniatura del hero. Click abre el detalle.

Detalle

Estructura vertical:

  1. Hero (imagen destacada, 16:9). Centrado en desktop con max-width. Botón "Cambiar" abre el media picker; "Quitar" vuelve a usar la primera de la galería como fallback.
  2. Galería:
    • Pulsa una foto para abrirla en MediaDetail.
    • Mantén pulsado 1 segundo en táctil (o arrastra con ratón) para reordenar.
    • ★ marca como hero (queda con borde dorado), ✕ quita de la galería.
    • "+ Añadir": MediaPicker para seleccionar más imágenes.
    • "💾 Guardar": al cambiar el orden, este botón aparece. Pulsa para persistir.
    • "▶ Slideshow": modo presentación con esta galería.
  3. Form + Configuración avanzada (lado a lado en desktop):
    • Título, resumen, estado (borrador/publicar/privado/pendiente).
    • Layout: st1 Column Grid Sidebar · st2 Carousel · st3 Full Width · st4 Slider.
    • Categorías (multi-selección).
    • Configuración avanzada del tema kotlis: campos dinámicos según el layout activo (sidebar, secciones de contenido, datos del proyecto, prev/next, etc.).
    • Vincular carpeta YZMF para sincronizar la galería con todas las imágenes de una carpeta.
  4. Acciones finales:
    • ↗ Ver portfolio (URL pública).
    • ⚙ Editor WP.
    • 📋 Duplicar como plantilla (con opción de copiar también la galería).
    • 🗑 Mover a papelera.

Categorías de portfolio

Pantalla independiente: Ajustes → Categorías de portfolio. CRUD completo con árbol jerárquico (igual que las carpetas).

9 · Galerías de cliente

Sistema completo para compartir selecciones privadas con clientes vía un enlace público con token único. Sin necesidad de cuenta de WordPress para el cliente.

Crear una galería

  1. Ajustes → Galerías de cliente → "+ Nueva galería".
  2. Pon un título (lo verá el cliente).
  3. Configura los campos:
    • Cliente: nombre + email (para notificaciones).
    • Mensaje: texto que aparece encima de la galería para el cliente.
    • Caducidad: fecha tras la cual el enlace deja de funcionar.
    • Contraseña: opcional. Si la pones, el cliente la introduce antes de ver.
    • Permisos: ⬇ permitir descarga · 💬 permitir comentarios.

Añadir imágenes

Tres formas:

  • + Añadir: MediaPicker — explora carpetas y selecciona.
  • ↻ Importar: bottomsheet con dos opciones — desde una carpeta YZMF (importa todas), o desde un portfolio existente. Filtra duplicados automáticamente.
  • Drag & drop: reordena (mantén pulsado 600ms en touch).
  • ✕ sobre cada thumb la quita de la galería (no la borra del catálogo).

Compartir el enlace

La URL es https://yezraelperez.es/g/token. Botón "📋" copia al portapapeles. El token usa caracteres URL-safe (base64): no se puede adivinar.

Tab "Actividad"

3 stats grandes: ★ favoritas · 💬 comentarios · 👁 visitas.

Log cronológico con cada acción del cliente (favorita o comentario), miniatura, fecha exacta, texto del comentario y link directo a MediaDetail.

Exportar selección del cliente

Cuando el cliente marca favoritas, las puedes exportar para procesarlas en Lightroom/Capture One:

  • 📄 Descargar .txt: una línea por filename original. Pegas en el filtro de Lightroom (Library Filter → Filename → contains any of) y tienes esas mismas fotos seleccionadas en tu catálogo.
  • 📊 Descargar .csv: con id, filename, título y comentarios (concatenados con |). Excel-friendly con BOM UTF-8.
  • 📋 Copiar lista: al portapapeles para pegado rápido.

Vista del cliente

El cliente abre /g/token en cualquier navegador. Ve un frontend standalone (sin Vue, JS vanilla) con:

  • Login con contraseña si la galería está protegida.
  • Grid responsive con todas las imágenes.
  • Lightbox con flechas, botón ★ favorita, contador, botón comentar.
  • Si tiene permiso, botón "⬇ Descargar favoritas" (descarga en lote).
  • "🔗 Copiar enlace" para reenviarlo.
Notificaciones por email Cada vez que el cliente marca una favorita o deja un comentario, recibes un email (configurable en wp-admin → Galerías de cliente → Ajustes).

10 · Mapa de ubicaciones

Mapa Leaflet con dos capas: ubicaciones manuales (CPT yzmf_location, p.ej. "Estudio Madrid") y fotos georreferenciadas (extraídas del EXIF o asignadas a mano).

Acciones

  • Click en mapa vacío: crear ubicación nueva con título + descripción.
  • Click en marcador existente: editar/eliminar.
  • Búsqueda: usa Nominatim (OpenStreetMap proxy) para encontrar direcciones por nombre.
  • Filtros: ver solo ubicaciones / solo fotos / ambas.

Histograma agregado de tus disparos. Útil para auto-conocimiento de tus hábitos:

15 · Seguridad y actividad de logins

Auto-cierre de sesión por inactividad

En Ajustes → Sesión, selector con 0 (nunca) · 5 min · 15 min · 30 min · 1h · 4h · 12h. Tras X min sin mover ratón ni pulsar teclas ni hacer scroll, sesión cerrada automáticamente.

Política de bloqueo de logins

3 parámetros configurables:

  • Intentos máximos: 5 por defecto. 0 desactiva el lockout.
  • Ventana: minutos en los que se cuentan los intentos. 15 por defecto.
  • Penalización: minutos de bloqueo. 30 por defecto.

Actividad de logins

  • 3 stats: logins recientes · intentos fallidos · IPs bloqueadas activas.
  • Lista de IPs bloqueadas con timer de desbloqueo y botón "Liberar bloqueos".
  • Historial completo (200 eventos) con fecha exacta, usuario, IP, navegador detectado.
Si te bloqueas a ti mismo Pulsa "Liberar bloqueos" desde otro dispositivo donde sigas dentro, o accede a wp-admin por el navegador (la cookie de sesión es independiente del Basic Auth).

16 · Integraciones avanzadas

Schema.org enrichment (siempre activo)

El plugin inyecta JSON-LD automático en <head> para portfolios, attachments y home page. Datos ricos: ImageObject con dimensiones, contentLocation/geo si está geolocalizado, keywords con tags IA, color con la paleta dominante, exifData con cámara/focal/ISO/etc.

Configurable en wp-admin (opciones yzmf_default_license, yzmf_author_name, yzmf_social_*). Verifica con Google Rich Results Test.

Lightroom Publish Service

Endpoints REST para integrar Adobe Lightroom Classic:

  • GET /yzmf/v1/lightroom/check — healthcheck.
  • GET /yzmf/v1/lightroom/folders — lista de carpetas YZMF disponibles.
  • POST /yzmf/v1/lightroom/publish — multipart con file + folder_id + metadata.

Tres estrategias de integración: plugin .lrplugin custom, plantilla XML-RPC nativa, o Python watcher de carpeta exportada (la más simple). Documentación detallada en docs/lightroom.md.

Hotlink Watermark

Plugin yzmf-hotlink-watermark. Cuando alguien enlaza directamente una de tus imágenes desde un dominio externo, el plugin reescribe la URL al vuelo y sirve una versión con marca de agua (Imagick + GD fallback) cacheada en disco.

Configurable en wp-admin → Settings → YZMF Hotlink:

  • Whitelist de dominios permitidos (incluye tu dominio por defecto).
  • Marca de agua: PNG con transparencia o texto con opacidad/posición.
  • Permitir motores de búsqueda y previsualizadores (Google, Facebook, etc.).
  • Botón "Vaciar caché" para regenerar todo tras cambiar la marca de agua.

18 · Resolución de problemas

"Sesión inválida — vuelve a iniciar sesión"

El cliente HTTP detecta 4 errores 401 consecutivos en menos de 8 segundos. Causas habituales:

  • La contraseña ha cambiado en wp-admin (cambia tu Application Password).
  • Tu IP está bloqueada por intentos fallidos (libera bloqueos en Ajustes o wp-admin).
  • El plugin yz-media-folders está desactivado en el servidor.

El Dashboard no muestra estadísticas

El endpoint /stats se cachea 5 minutos. Tras subir muchas fotos, espera o haz pull-to-refresh. Si aún así no aparecen, asegúrate de que la última versión del plugin esté desplegada (con uploads_365d y los counts de salud).

El logo nuevo no aparece

El brand store cachea en localStorage para evitar parpadeo. Pull-to-refresh en Dashboard o cierra/abre la app para ver el cambio.

Galería de cliente devuelve 404 al abrir

  • WP Admin → Settings → Permalinks → Guardar (sin cambios). Esto fuerza el flush de rewrite rules.
  • Comprueba que Permalinks no es "Plain" — necesita estructura tipo /%postname%/.
  • El plugin tiene auto-flush por versión y un fallback en parse_request, pero algunos sistemas de caché agresiva (LiteSpeed) pueden necesitar purga manual.

El drag & drop de la galería no inicia

En táctil, requiere mantener pulsado 1 segundo antes de empezar a arrastrar. Esto es intencional para no romper el scroll. En desktop con ratón el drag es inmediato.

Las fotos no aparecen al cliente en una galería protegida

  • Verifica en Ajustes → Galerías de cliente → la galería en cuestión que no haya caducado.
  • El sistema usa sessionStorage del navegador del cliente. Si el cliente borra cookies/storage, tendrá que volver a meter la contraseña.

Quiero forzar regeneración de paletas/tags retroactivos

No hay UI todavía. Las imágenes nuevas (subidas tras la activación de cada feature) las reciben automáticamente. Para retroactivos, se puede hacer un script de backfill bajo demanda — pídelo y se monta como endpoint admin.
